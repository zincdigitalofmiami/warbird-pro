# v7 Pine Interface: Surface Divergences

**Date:** 2026-04-12
**Updated:** 2026-04-26
**Status:** Active — documents intentional mechanical divergences between the two v7 files.

## Institutional (`v7-warbird-institutional.pine`) — Live Chart Surface

- **Entry trigger:** `acceptEvent` path (zone break → retest → close back through zone)
- **Purpose:** human-readable signal overlay, alert broadcast, live entry cue
- **Output budget:** 58/64 (55 plot + 3 alertcondition, 6 headroom)
- **Alerts:** 3 `alertcondition()` calls (ENTRY LONG, ENTRY SHORT, PIVOT BREAK)

## Strategy (`v7-warbird-strategy.pine`) — Strategy Tester / AG Export Surface

- **Entry trigger:** gated SETUP -> `acceptEvent` + confirmation path, with optional footprint scalp path and risk/ladder/HTF suppressors
- **Purpose:** Strategy Tester execution surface and Pine `ml_*` export-compatibility mirror. Canonical local AG rows are generated by `scripts/ag/build_ag_pipeline.py`, not by TradingView strategy orders.
- **Output budget:** 60/64 (60 plot, 4 headroom); uses `alert()` not `alertcondition()`
- **Exit:** SL or active target; TP1 by default with fast-runner target promotion where configured
- **Commission floor:** $1.00/side; `use_bar_magnifier=true`; `slippage=1` — pinned in `strategy()` declaration

## Shared Contract (must stay in sync)

- All `ml_*` hidden export field names must match between both files
- `stopFamilyCode`, `eventPivotInteractionCode`, `setupArchetypeCode` patterns identical
- All coupled input defaults must match (strategy is source of truth)
- Verified by `scripts/guards/check-indicator-strategy-parity.sh`

## Intentionally NOT Enforced by Guard

- Exit mechanics (alert-based lifecycle vs. SL/TP5 only)
- `alertcondition` presence (institutional has 3, strategy has 0)
- Strategy-only `strategy()` declaration with Bar Magnifier / commission / slippage pins

## Closure Note: candidateSetup Divergence (reconciled 2026-04-26)

A prior entry in this doc claimed the strategy used a `candidateSetup` path (`priceAtFibLevel` — any close within 15% ATR of 6 fib levels) while the indicator used `acceptEvent`. This was stale: the live strategy already followed the gated `acceptEvent` / confirmation path into `TRADE_SETUP` -> `TRADE_ACTIVE`. The `candidateSetup` variable is still computed for diagnostic/export purposes but does not drive entries. Therefore the strategy is not an "every fib touch" local training generator; that role belongs to the Python reconstruction pipeline.

## Rationale

The institutional file is optimized for human operator use: it shows high-confidence accept signals on the chart. The strategy file is optimized for TradingView execution testing and parity with the live Pine export surface. Local AutoGluon coverage comes from warehouse reconstruction of fib interactions and stop variants, where AG decides quality from point-in-time features rather than from Pine order frequency.
